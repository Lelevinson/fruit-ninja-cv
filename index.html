Slide 3 · Why Speed Matters

Comic Story: How One Swipe Is Judged

Analogy: like a sports referee. The game watches your move, checks rules, then gives a decision.

Frame 1: Swipe captured

Game records where your blade moved.

→

Frame 2: Two checks

  • Did the path pass close to fruit?
  • Was the swipe fast enough?

Both checks must pass in the same moment.

→

Frame 3: Final decision

Slow swipe: no cut
Fast swipe: slice

That is why fast motion feels more rewarding.

Check 1: Touch path

The swipe path should pass close enough to the fruit area.

Check 2: Speed gate

The swipe must be fast enough to feel like a real cut.

Student tip

Use a clear diagonal swipe instead of tiny hand shakes.

Teacher note

This teaches threshold-based decisions in real-time systems.

True code rule from main.py: the game checks a minimum cut speed (MIN_CUT_VELOCITY = 80).
Slide 4 · Hit Detection

How the Game Decides "Hit" or "Miss"

Analogy: imagine drawing a thick marker line. If the fruit circle touches it, that is a hit.

1. Draw fruit as a circle.
2. Draw blade path as a thick line.
3. Check shortest distance between them.
4. If close enough, count it as slice.
Friendly shortcut

If thick blade paint touches the fruit bubble, score goes up.

Real math behind it

Code measures nearest distance between blade line and fruit center.

Slide 5 · Repeating Recipe

What the Game Repeats Every Moment

Analogy: this is a cooking recipe the game repeats 60 times each second.

ListenVery quick
InputQuick read
UpdateMove objects
Hit checkSlice or miss
DrawMost visual work
SoundFeedback layer

1) Listen

Check click, key press, and exit events.

2) Read Input

Get finger or mouse position and speed.

3) Move Objects

Spawn fruit, apply gravity, update positions.

4) Check Hits

Test blade path against fruit circles.

5) Draw Scene

Show camera, fruit, blade, effects, and score.

6) Play Sound

Splat, bomb, combo, then show the frame.

Technical truth in simple words: 60 FPS means this loop runs 60 times per second.
Why this matters

If this loop slows down, game feels laggy and less fun.

What keeps it smooth

Short tasks per frame and clean object updates each cycle.

Slide 6 · Game Challenges

Classic Mode vs Survival Mode

Analogy: Classic is practice with chances. Survival is one-life challenge.

Classic Mode

  • Start with 3 lives.
  • Slice fruit: +1 score.
  • Hit bomb: lose 1 life.
  • Miss fruit: lose 1 life.

Survival Mode

  • Start with 1 life only.
  • Slice fruit: +1 score.
  • Hit bomb: game over.
  • Miss fruit: game over.
Best for beginners

Classic mode gives extra chances while learning hand control.

Best for challenge

Survival mode is high pressure and rewards clean focus.

Same core skill

Both modes still depend on swipe accuracy and speed timing.

Slide 7 · Meet the Team

Each Python File Has One Job

Analogy: this project works like a school event team where each person has one role.

main.py

The captain: runs the game loop and scene flow.

input_manager.py

The translator: converts mouse or hand motion to blade motion.

sensors.py

The camera helper: reads webcam and tracks fingertip smoothly.

physics.py

The referee: decides whether blade and fruit touched.

game_engine.py

The rule book: score, lives, miss, bomb, game over.

game_objects.py

The actors: fruit, bomb, blade, splash, explosion objects.

ui_manager.py

The stage crew: menu, mode select, pause, game-over screens.

audio_manager.py

The DJ: music and sound effects for player feedback.

Big lesson for students

Good projects split work into modules so code is easier to build and debug.

How to explain it simply

One file = one main responsibility, just like one student role in a team.

Slide 8 · Recap

Quick Myth Check + Final Takeaway

You now know this game is not random magic. It is camera + math + rules + design.

Myth: "The game reads one photo." Truth: It reads a video stream frame by frame.
Myth: "Any movement can slice." Truth: Speed must pass the minimum cut threshold.
Myth: "Hit detection is random." Truth: It uses geometry: blade path vs fruit circle.
Myth: "Everything is one big file." Truth: Many files work together like a team.
Final takeaway: this project is a great example of combining computer vision, game design, and beginner-friendly Python structure.
